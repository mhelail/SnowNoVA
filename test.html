Cross-Site scripting (XSS)


Açıklama: Siteler arası komut dosyası çalıştırma (XSS) saldırganların web sayfalarına tehlikli betikler enjekte etmelerine olanak tanıyan bir istemci tarafı web güvenlik zafiyetidir.


XSS Saldırıları:


  • XSS zafiyeti genellikle web uygulamalarında girdi doğrulama eksikliğinden kaynaklanır.
  • Saldırganlar, XSS zafiyetlerini zararlı kodları web uygulamalarına enjekte etmek için kullanırlar. XSS, istemci tarafı bir zafiyet olduğundan, bu betikler kurbanın tarayıcısı tarafından yürütülür.
  • XSS zafiyetleri, giriş doğrulamasından yoksun olan ve Javascript, Flash, CSS gibi istemci tarafı betik dillerini kullanan web uygulamalarını etkiler


XSS kategorileri:


  1. Stored XSS
  2. Reflected XSS
  3. DOM Tabanlı XSS


1. Stored XSS


bir saldırganın, doğrulanmamış bir giriş aracılığıyla web uygulamasının veritabanına veya kaynak koduna JavaScript kodu enjekte edebildiği bir zafiyettir.

- Örneğin, bir saldırganın uygun doğrulama yapılmadan bir web sayfasına zararlı bir XSS yükü enjekte etmesi durumunda, bu web sayfasına ziyaret eden herhangi bir kişinin tarayıcısı tarafından enjekte edilen XSS yükü yürütülecektir.


2. Reflected XSS


Reflected/non-persistent cross-site scripting XSS'in en yaygın şeklidir ve bir kurbanı, saldırıya açık bir web sitesine özel olarak oluşturulmuş bir bağlantıyı (XSS yüküyle) tıklamaya kandırma işlemi içerir.

- Örneğin, Kurban bağlantıya tıkladığında, web sitesi yanıtının bir parçası olarak XSS yükünü kurbanın tarayıcısına dahil eder ve yük çalıştırılır..


Potansiyel Etki:


Saldırgan, potansiyel kurbanlara JavaScript yükü içeren bağlantılar gönderebilir veya bunları başka bir web sitesinde bir iframe'e gömebilir, böylece kurbanları tarayıcılarında kod çalıştırmaya ikna edebilir ve potansiyel olarak oturum veya müşteri bilgilerini ifşa edebilir.


XSS simülatörü

Simülatörü denemek için butona tıklayın.



Scenario:


Senaryomuzda, web sayfasında kullanıcıları isimlerini girmeye davet eden bir metin formu bulunmaktadır. Gönderildikten sonra, web sitesi "Hoş geldiniz, [kullanıcının adı]" gibi bir karşılama mesajı görüntüler.


Zafiyet:


XSS zafiyeti, web sitesinin kullanıcı girdisini sayfada görüntülemeden önce doğru bir şekilde doğrulaması gerektiği için meydana gelir. Bir kullanıcı ismini forma girdiğinde, web sitesi bu girdiyi doğrudan karşılama mesajına gömerek, potansiyel olarak kötü amaçlı içerikler için herhangi bir kontrol yapmadan işler.


Sömürü:


Bir saldırgan, formda kötü amaçlı JavaScript kodu içeren bir isim gireerek bu zaafiyeti sömürebilir. Örneğin, "Muhammed" gibi düzenli bir isim yerine, saldırgan gibi bir şey girebilir 

"<script>alert("XSS_SnowNoVA");</script>".


Uygulama:


Web sitesi karşılama mesajını görüntülerken, saldırganın girdisini HTML içeriğinin bir parçası olarak ekler. Sonuç olarak, tarayıcı, isim alanına gömülü olan JavaScript kodunu yorumlar ve çalıştırır. Bu durumda, bir uyarı ile "XSS_SnowNoVA" mesajını görüntüler.


Sonuçlar:


Bu zafiyet, saldırganların diğer kullanıcıların tarayıcılarının bağlamında keyfi JavaScript kodu enjekte etmelerine ve çalıştırmalarına olanak tanır. Saldırganın niyetine bağlı olarak, bu, oturum cookie çalma, kullanıcıları sahte web sitelerine yönlendirme veya kullanıcı hesaplarını ele geçirme gibi çeşitli kötü amaçlı eylemlere yol açabilir.
