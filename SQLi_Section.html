SQLi (SQL Injection)


Açıklama: SQL enjeksiyonu (SQLi), bir saldırganın zararlı SQL ifadelerini bir uygulamanın giriş alanlarına enjekte ettiği bir web uygulaması enjeksiyon zafiyetidir.


SQL Enjeksiyonuna Giriş:



  • SQL enjeksiyonu (SQLi), bir web uygulamasının kullanıcı girdilerini doğru bir şekilde doğrulamaması durumunda meydana gelir; bu durum, saldırganın veritabanını manipüle edebilecek veya hassas bilgilere erişebilecek SQL kodları/sorguları enjekte etmesine izin verir.


SQLi kategorileri:


  1. In-Band SQLi
  2. Blind SQLi
  3. Out-of-Band SQLi


1. In-Band SQLi


In-band SQL enjeksiyonunda, saldırgan kötü amaçlı SQL kodunu web uygulamasına enjekte eder ve saldırının sonuçlarını kodun gönderildiği aynı kanal üzerinden alır.


In-band SQL SQL enjeksiyon saldırıları tehlikelidir çünkü hassas bilgileri çalmak, verileri değiştirmek veya silmek, hatta tüm web uygulamasını veya tüm sunucuyu ele geçirmek için kullanılabilir.

2. Blind SQLi


Blind SQL Injection, saldırganın enjekte edilen SQL query sonuçları veya veritabanı hakkında doğrudan bilgi vermeyen bir web uygulamasındaki bir açığı kullanabildiği bir SQL Injection saldırı türüdür.

Bu tür bir saldırıda, saldırgan kötü amaçlı SQL kodunu uygulamanın giriş alanına enjekte eder, ancak uygulama yanıtta saldırgana herhangi bir faydalı bilgi veya hata mesajı döndürmez.


Potansiyel Etki:


SQL enjeksiyon saldırıları, bir veritabanında saklanan hassas verilere yetkisiz erişim sağlayabilir, bu da saldırganların müşteri verileri, finansal bilgiler ve fikri mülkiyet gibi gizli bilgileri görüntülemesine veya çalmasına olanak tanır. Ayrıca, bu saldırılar veritabanında saklanan verilerin manipülasyonuna, yani verilerin değiştirilmesine veya silinmesine yol açabilir, bu da veri kaybı veya bozulmasına neden olabilir. Eğer bir veritabanı kullanıcısının yönetici ayrıcalıkları varsa, saldırgan hedef sisteme zararlı kod kullanarak erişim sağlayabilir.


SQLi simülatörü

Senaryo:


Senaryomuzda, simülatör interaktif bir giriş formuna sahip bir giriş sayfası özelliği taşır ve bu form, kullanıcı girdilerine bağlı olarak dinamik olarak SQL Sorgularını ve SQL Sonuçlarını gösterir.


Zafiyet:


SQL enjeksiyonu zafiyeti, bir web uygulamasının kullanıcı girdilerini uygun şekilde doğrulamadığı veya sterilize etmediği durumlarda meydana gelir. Bu tür zafiyetler, saldırganların güvenli olmayan kullanıcı girdilerini kullanarak SQL sorgularını manipüle etmelerine olanak tanır.


Sömürü:


Bir kullanıcı adı ve şifre giriş formunda, saldırgan standart veriler yerine SQL ifadeleri girerek veritabanı sorgularını değiştirebilir.


Uygulama:


Kullanıcı adı veya şifre alanına SQL komutları ekleyerek, saldırganlar veritabanı sorgusunu manipüle edebilir.

Örneğin, 

' OR '1'='1
kullanıcı adı alanına yazmak, her zaman doğru olan bir koşul ekler ve bu sorgunun tüm kullanıcıları döndürmesine neden olur.






Saldırgan, SQL enjeksiyonu yoluyla tüm kullanıcı bilgilerini elde edebilir. Bu, özellikle kullanıcı adları ve şifreler gibi hassas bilgileri içeren tablolar için ciddi bir güvenlik riski oluşturur.

Sonuçlar:


Bu zafiyet, saldırganların güvenli olmayan kullanıcı girdilerini kullanarak SQL sorgularını manipüle etmelerine olanak tanır. Saldırganın niyetine bağlı olarak, bu durum veritabanından hassas bilgilerin çalınması, veritabanı sorgularının değiştirilmesi veya veritabanı tablolarının silinmesi gibi çeşitli kötü amaçlı eylemlere yol açabilir.
